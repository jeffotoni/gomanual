Fiber - Error Handling

É essencial garantir que o Fiber detecte todos os erros que ocorrem durante a execução de handlers e middlewares.

Você deve retorná-los à função do handler, onde o Fiber os capturará e processará.

Fiber não lida com panics por padrão. Para recuperar um panic gerado por qualquer handler na pilha, você precisa incluir o middleware Recover.

Você pode usar a estrutura de erro personalizada do Fiber para passar um status code adicional usando fiber.NewError().

É opcional passar uma mensagem; se for deixado em branco, o padrão será a mensagem de código de status (Ex: 404 é igual a "Not Found").

fiber.go

package main

import (
  "github.com/gofiber/fiber/v2"
  "github.com/gofiber/fiber/v2/middleware/recover"
)

func main() {
    app := fiber.New()

    app.Use(recover.New())

    app.Get("/", func(c *fiber.Ctx) error {
      return fiber.NewError(fiber.StatusServiceUnavailable, "Error!")
    })

    app.Listen(":3000")
}
