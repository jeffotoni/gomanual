Generics - Quando usar?

Um caso é ao escrever funções que operam nos container-types definidos pela linguagem: slices, maps e channels. Se uma função tiver parâmetros com esses tipos e o código da função não fizer nenhuma suposição específica sobre os tipos dos elementos, pode ser útil usar um type-parameter.

Outro caso em que os type-parameters podem ser úteis é para estruturas de dados de uso geral. Uma estrutura de dados de propósito geral é algo como um slice ou um map, mas que não está embutido na linguagem, como uma linked-list ou uma árvore binária.

Hoje, os programas que precisam dessas estruturas de dados normalmente fazem uma de duas coisas: escrevem com um tipo específico ou usam um interface-type. Substituir um tipo específico por um type-parameter pode produzir uma estrutura de dados mais geral que pode ser usada em outras partes do programa ou por outros programas. Substituir um interface-type por um type-parameter pode permitir que os dados sejam armazenados de forma eficiente, economizando recursos de memória; ele também pode permitir que o código evite declarações de tipo e seja totalmente verificado em tempo de compilação.
